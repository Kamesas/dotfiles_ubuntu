HOW IT WORKS:
-------------

GNU Stow creates SYMLINKS from ~/dotfiles/ to your home directory.

BEFORE:
  ~/dotfiles/bash/.bashrc    (file)
  ~/.bashrc                  (file)

AFTER STOW:
  ~/dotfiles/bash/.bashrc    (actual file - edit this!)
  ~/.bashrc                  → symlink to ~/dotfiles/bash/.bashrc

BENEFITS:
  ✓ Edit files in ~/dotfiles/ - changes instantly reflected in ~
  ✓ All configs in one Git repository
  ✓ Easy to sync across multiple machines
  ✓ Automatic backups before changes

BACKUP SYSTEM:
--------------

Create backup:     ./backup.sh create
List backups:      ./backup.sh list
Restore:           ./backup.sh restore backup_20240101_120000
Clean old:         ./backup.sh clean 10

Backups stored in: ~/.dotfiles-backups/

PACKAGE STRUCTURE:
------------------

Each package follows this pattern:

  package-name/
  ├── .config/           # For files that go in ~/.config/
  │   └── app/
  │       └── config
  └── .dotfile           # For files that go in ~/

Example - bash package:
  bash/
  ├── .bashrc
  ├── .bash_logout
  └── .profile

Example - nvim package:
  nvim/
  └── .config/
      └── nvim/
          ├── init.lua
          └── lua/

TROUBLESHOOTING:
----------------

Conflicts during stow:
  $ ./backup.sh create
  $ rm ~/.bashrc
  $ stow bash

Broken symlinks:
  $ cd ~/dotfiles
  $ stow -R bash

Undo everything:
  $ ./backup.sh restore backup_YYYYMMDD_HHMMSS
  OR
  $ ./install.sh uninstall
